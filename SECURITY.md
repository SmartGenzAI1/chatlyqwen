
# Security Policy

## Supported Versions

We only support the latest version of Chatly. Security updates are only provided for the current version.

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability in Chatly, please report it to us immediately.

### How to Report

1. **Email**: security@chatly.app (preferred method)
2. **GitHub Security Advisory**: Create a private security advisory

### What to Include

- Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting)
- Full paths of source files related to the issue
- Location of affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit it

### Response Time

We aim to respond to all security reports within 24-48 hours. Critical issues will be addressed immediately.

## Security Updates

Security updates will be released as soon as possible after a vulnerability is confirmed. We will:

1. Fix the vulnerability
2. Release a new version with the fix
3. Document the security issue in the release notes
4. Notify affected users if necessary

## Bug Bounty

We don't currently have a formal bug bounty program, but we may reward significant security findings at our discretion.

## Public Disclosure Policy

We believe responsible disclosure is important. Please give us a reasonable time to fix security issues before public disclosure. We typically ask for 30-90 days depending on the severity.

## Contact

For security issues only: security@chatly.app
For general inquiries: support@chatly.app
